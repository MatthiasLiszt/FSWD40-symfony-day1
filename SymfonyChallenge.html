Create a new Symfony Project

As you know from the previous material, to generate a new Symfony project you need to navigate through the terminal (Command prompt) to C:\xampp\htdocs and then you need to type this command (in our case we will name our project “todolist”):

symfony new todolist 3.4

When you run this command you will see a result as in the image below. This command will generate new Symfony project (folder) inside htdocs folder.

You have two ways of opening your project :

  1. http:\\localhost\todolist\web\app_dev.php
  2. open the terminal or command prompt inside your project folder and run this code :

php bin/console server:run

Now you can open your project folder in Sublime and we are ready to start our todolist app development in Symfony.

Symfony Project: todolist

Create a new Controller

Go to the src folder and open AppBundle -> Controller and create a new controller (file: TodoController.php) inside the  Controller folder. Now you can open TodoController.php and insert this code inside it:

<?php

namespace AppBundle\Controller;

use Sensio\Bundle\FrameworkExtraBundle\Configuration\Route;

use Symfony\Bundle\FrameworkBundle\Controller\Controller;

use Symfony\Component\HttpFoundation\Request;

class TodoController extends Controller

{

    /**

     * @Route("/todos", name="todo_list")

     */

    public function listAction(Request $request)

    {

        return $this->render('todo/index.html.twig');

    }

}

As you can see in TodoController.php we need to have index.html.twig view inside the todo folder which you need to create, so let’s create it. Go to app -> Resources -> views and create a new folder todo. Inside folder todo, create a file index.html.twig. Inside this file for now you can just type a word INDEX and save it. Now you can check http://127.0.0.1:8000/todos you should get INDEX word as a result in your browser. If you can see the result INDEX in your browser it means that you can go further. So let’s create three  more files inside the todo folder:

  • create.html.twig (type CREATE inside)
  • details.html.twig (type DETAILS inside)
  • edit.html.twig (type EDIT inside)

Now we need to change TodoController.php file, basically we need to add an action for every view that we’ve created:

<?php

use Sensio\Bundle\FrameworkExtraBundle\Configuration\Route;

use Symfony\Bundle\FrameworkBundle\Controller\Controller;

use Symfony\Component\HttpFoundation\Request;

{

    /**

     * @Route("/todos", name="todo_list")

     */

    public function listAction()

    {

        return $this->render('todo/index.html.twig');

    }

        /**

     * @Route("/todo/create", name="todo_create")

     */

    public function createAction(Request $request)

    {

        return $this->render('todo/create.html.twig');

    }

            /**

     * @Route("/todo/details/{id}", name="todo_details")

     */

    public function detailsAction($id)

    {

        return $this->render('todo/details.html.twig');

    }

   

            /**

     * @Route("/todo/edit/{id}", name="todo_edit")

     */

    {

        return $this->render('todo/edit.html.twig');

    }

}

Check if you get the right results for these requests:

http://127.0.0.1:8000/todo/create

http://127.0.0.1:8000/todo/details/1

If you get the expected results (INDEX, CREATE. DETAILS, EDIT) that means all your actions are now linked to your views and we can go to the next step - layout implementation.

Doctrine

More information about the Symfony Doctrine you can find in the Symfony documentation page on this link: https://symfony.com/doc/3.3/doctrine.html

Database create

First step is to open parameters.yml file in the app->config folder. This file keeps information about your database connection. Insert database_name and other parameters if you are not using the default settings for XAMPP.

# This file is auto-generated during the composer install

parameters:

    database_host: 127.0.0.1

    database_port: null

    database_name: todo_list

    database_user: root

    database_password: null

    mailer_transport: smtp

    mailer_host: 127.0.0.1

    mailer_user: null

    mailer_password: null

    secret: ed436fd8e7560170bc6025814ee9139affb2b4f2

After this you need to open Command prompt and to navigate to your project folder. Now we will run doctrine commands. First command will be a command for the database creation, this command will use data from parameters.yml file and generate the database for us (you can check in phpmyadmin):

php bin/console doctrine:database:create

 

Creating an Entity Class

To create an entity we need to run the following command, if you want to know more about this process you can read it on this link in the official Symfony documentation

 https://symfony.com/doc/3.4/doctrine.html#creating-an-entity-class

php bin/console doctrine:generate:entity

After you run the command to generate an  entity you will get this message:

Welcome to the Doctrine2 entity generator

This command helps you generate Doctrine2 entities.

First, you need to give the entity name you want to generate.

You must use the shortcut notation like AcmeBlogBundle:Post.

The Entity shortcut name:

Now you need to insert this for the Entity shortcut name: AppBundle:Todo

After that you will get this message:

Configuration format (yml, xml, php, or annotation) [annotation]:

Just press Enter this will keep the default annotation format. Now you can start to add fields, first we will type: name and press enter, we will keep the default values for field type, length, nullable, unique - just press enter which means the default value. We will also create fields: category, description, priority with the default values, the last two fields are: due_date and create_date with the type datetime. At the end just press enter and you should get the message:

Everything is OK! Now get to work :).

 

You can check inside src folder -> AppBundle -> Entity -> todo.php. In this file every property that we generated using doctrine has getters and setters. Next step will be to generate the table with these properties.

Creating the Database Tables/Schema

We need to run this command to generate a table with the properties that we’ve already defined:

php bin/console doctrine:schema:update --force

 

At the end you should get this message:

Updating database schema...

Database schema updated successfully! "1" query was executed

Now if you open phpmyadmin you should be able to see the todo table like in the image below:

Fetching data from the database

You need to change a few files to fetch data from the database. First you need to change your TodoController.php to this:

<?php

namespace AppBundle\Controller;

use AppBundle\Entity\Todo;

use Sensio\Bundle\FrameworkExtraBundle\Configuration\Route;

use Symfony\Bundle\FrameworkBundle\Controller\Controller;

use Symfony\Component\HttpFoundation\Request;

use Symfony\Component\Form\Extension\Core\Type\TextType;

use Symfony\Component\Form\Extension\Core\Type\TextareaType;

use Symfony\Component\Form\Extension\Core\Type\DateTimeType;

use Symfony\Component\Form\Extension\Core\Type\ChoiceType;

use Symfony\Component\Form\Extension\Core\Type\SubmitType;

class TodoController extends Controller

{

    /**

     * @Route("/", name="todo_list")

     */

    public function listAction(){

        $todos = $this->getDoctrine()->getRepository('AppBundle:Todo')->findAll();

        // replace this example code with whatever you need

        return $this->render('todo/index.html.twig', array('todos'=>$todos));

    }

     /**

     * @Route("/todo/create", name="todo_create")

     */

    public function createAction(Request $request){

        $todo = new Todo;

        $form = $this->createFormBuilder($todo)->add('name', TextType::class, array('attr' => array('class'=> 'form-control', 'style'=>'margin-bottom:15px')))

        ->add('category', TextType::class, array('attr' => array('class'=> 'form-control', 'style'=>'margin-bottom:15px')))

        ->add('description', TextareaType::class, array('attr' => array('class'=> 'form-control', 'style'=>'margin-bottom:15px')))

        ->add('priority', ChoiceType::class, array('choices'=>array('Low'=>'Low', 'Normal'=>'Normal', 'High'=>'High'),'attr' => array('class'=> 'form-control', 'style'=>'margin-botton:15px')))

    ->add('due_date', DateTimeType::class, array('attr' => array('style'=>'margin-bottom:15px')))

    ->add('save', SubmitType::class, array('label'=> 'Create Todo', 'attr' => array('class'=> 'btn-primary', 'style'=>'margin-bottom:15px')))

        ->getForm();

        $form->handleRequest($request);

        if($form->isSubmitted() && $form->isValid()){

            //fetching data

            $name = $form['name']->getData();

            $category = $form['category']->getData();

            $description = $form['description']->getData();

            $priority = $form['priority']->getData();

            $due_date = $form['due_date']->getData();

            $now = new\DateTime('now');

            $todo->setName($name);

            $todo->setCategory($category);

            $todo->setDescription($description);

            $todo->setPriority($priority);

            $todo->setDueDate($due_date);

            $todo->setCreateDate($now);

            $em = $this->getDoctrine()->getManager();

            $em->persist($todo);

            $em->flush();

            $this->addFlash(

                    'notice',

                    'Todo Added'

                    );

            return $this->redirectToRoute('todo_list');

        }

        // replace this example code with whatever you need

        return $this->render('todo/create.html.twig', array('form' => $form->createView()));

    }

     /**

     * @Route("/todo/edit/{id}", name="todo_edit")

     */

    public function editAction( $id, Request $request){

        // replace this example code with whatever you need

        return $this->render('todo/edit.html.twig');

    }

     /**

     * @Route("/todo/details/{id}", name="todo_details")

     */

    public function detailsAction($id){

        // replace this example code with whatever you need

        return $this->render('todo/details.html.twig');

    }

}

Delete action

To delete we need to define a new action in TodoController.php like this:

    /**

     * @Route("/todo/delete/{id}", name="todo_delete")

     */

    public function deleteAction($id){

                 $em = $this->getDoctrine()->getManager();

            $todo = $em->getRepository('AppBundle:Todo')->find($id);

            $em->remove($todo);

             $em->flush();

            $this->addFlash(

                    'notice',

                    'Todo Removed'

                    );

             return $this->redirectToRoute('todo_list');

    }
